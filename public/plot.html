6: Gráficos al estilo Old School

Suponte que estas en la decada de los 80. Los ordenadores acaban de entrar en el mercado doméstico y a has decidido adquirir uno. Lucía, tu prima pequeña, acaba de empezar a estudiar las ecuaciones de la recta y tu como buen primo que eres has decidido intentar ayudarla.

La mejor forma de entender la ecuación de una recta (y = mx + n) es dibujandola, para ello vamos a hacer uso de nuestro nuevisimo ordenador. El problema es que en la decada de los 80 unicamente disponíamos del terminal y no teniamos ninguna herramienta del estilo de Matlab. Por ello te has decidido a crear un programilla que representa las gráficas en el propio terminal.

El programa recibirá como primer parámetro el mayor valor de la X que debemos representar (bajo ningún caso será de tipo negativo) y como segundo parámetro la ecuanción de la recta concreta, es decir que la m y la n no serán incognitas. Con estos parametros dibujaremos unos X e Y centrados en la pantalla. Hay que tener encuenta que los espacios en el terminal no són variables así que los ejes tendrán que estar escalados acordes a la ecuación a representar. La forma de escalar los ejes es mediante sintaxis, así un '#' en un eje represnta una unidad y un '-' una parte de una unidad. Así si la ecuación a representar fuese "y=5x" por cada '#' en el eje Y en el eje X tendriamos una cadena de 5 caracteres '----#' en la que únicamente el último representará la unidad. La ecuación de entrada siempre tendrá al menos una 'y' un '=' y una 'x' adicionalmente podría tener un valor concreto para la 'm' y la 'n' (siempre que se indiquen será mayor de 0), no se dejarán espacios entre los elementos de la cadena de texto que representa la ecuación. Para representar la recta se usarán las barras laterales '/' o '\'. La representación siempre se hará en el menor espacio posible de pantalla y siempre prevalecerá la representación de los ejes frente a la recta en caso de corte.

Ejemplo de input 


3 
y=5x+3

Guia del campeonato

Probar tu código:

Recuerda que debes tener instalado Nodejs. Una vez instalado haz lo siguiente:
  1. Abre la consola/terminal. En windows puedes darle a Inicio, teclear cmd y pulsar Enter.
  2. Ahora copia/pega este comando: 
    npm install seminarjs-contestclient -g

Para escribir tu solución al enunciado haz una carpeta en tu ordenador, y guarda dentro un archivo llamado por ejemplo ej1.js. Ahora vuelve a abrir la consola/terminal (instrucciones más arriba) y vete al directorio donde tienes el archivo, si estuviese en Documentos/Ejercicios harías cd Documentos/Ejercicios.

Finalmente ejecuta node ej1.js para ejecutar tu código en la consola. Para ver cómo leer el input del programa baja al paso "Programar la solución"

Enviar la solución:

Cuando tengas la solución lista y quieras probarla definitivamente, ejecuta el siguiente comando en la misma consola y directorio. Donde pone token tienes que poner tu correo electrónico hasta la @ sin incluirla.

seminarjsContest contest.nichejs.com [token] [archivo.js]

No pongas los [], son para indicarte donde tienes que rellenar.

Programar la solución:

Para leer el input puedes usar el código a continuación, en el bucle donde esta el comentario dispones de la variable line que contiene el valor de esa linea.

var readline = require('readline');
var rl = readline.createInterface({
  input: process.stdin,
  output: process.stdout
});

rl.on('line', function (line) {
  // Codigo aqui
});

Para mostrar las respuestas para cada linea usa console.log();

Si quieres programarlo en una función puedes usar lo siguiente:

var readline = require('readline');
var rl = readline.createInterface({
  input: process.stdin,
  output: process.stdout
});

rl.on('line', function (line) {
  solucion(line);
});


function solucion(line){
  // Aqui tu solucion, que usa una linea de codigo
}
